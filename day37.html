Introduction

In the previous lecture, we learned how to handle missing values in numerical features using Mean Imputation and Median Imputation.

In this lecture, the focus shifts to categorical (text) features.

Unlike numerical data, categorical variables cannot be filled using the mean or median. Therefore, different techniques are required.

Two Main Techniques Covered

1. Most Frequent (Mode) Imputation 2. Missing Category Imputation

Other techniques (Random Value Imputation, etc.) will be covered later.

Why Missing Values Matter

Almost every machine learning algorithm struggles with missing data. Therefore, before training a model, missing values must be handled appropriately.

The choice depends on:

Percentage of missing values

Type of feature

Numerical or Categorical

Missing mechanism

MCAR, MAR, MNAR

Recap — Numerical Imputation

For numerical columns, if missing values are less than 5%, generally use:

Mean Imputation Median Imputation

because these preserve the distribution better.

Technically, Yes. Every numerical column also has a mode.

However, this is rarely preferred because:

  • Mean performs better for symmetric distributions
  • Median performs better for skewed distributions

Mode is mainly used for categorical features.

Why Does This Help?

Instead of pretending we know the value, we explicitly tell the algorithm:

"This value was originally missing."

Sometimes, the fact that a value is missing itself carries useful information. Many machine learning models can learn patterns from this.

Numerical Analogy

In numerical features, sometimes people replace missing values with:

-999 999 99

These numbers indicate "This value was missing."

For Numerical

-999

For Categorical

"Missing"

When Should It Be Used?

Condition 1

Missing values are NOT random

MAR or MNAR

Condition 2

Missing percentage is high

40% 50% 60%

Advantages

  • Very easy to implement
  • Fast
  • Works well with many ML algorithms
  • Preserves information that a value was missing

Limitation

It does not actually estimate the original value.

It simply informs the model "This observation was missing." So performance may not always be optimal.

Scikit-Learn — Most Frequent

from sklearn.impute import SimpleImputer

imputer = SimpleImputer(strategy='most_frequent')

X_train = imputer.fit_transform(X_train)
X_test = imputer.transform(X_test)

fit_transform()

Learns the most frequent category from training data and applies it

transform()

Replaces missing values in both training and testing sets using the learned mode

Scikit-Learn — Missing Category

from sklearn.impute import SimpleImputer

imputer = SimpleImputer(
  strategy='constant',
  fill_value='Missing'
)

X_train = imputer.fit_transform(X_train)
X_test = imputer.transform(X_test)

Every missing value becomes "Missing"

Decision Guide

Use Mode Imputation when:

  • Missing values are low (typically <5–10%)
  • Missingness is approximately MCAR
  • One category clearly dominates
  • You want a simple and fast solution

Use Missing Category when:

  • Missing values are high
  • Missingness is informative (MAR/MNAR)
  • You want the model to recognize that a value was originally missing
  • There is no dominant category

Important Interview Questions

Why can't we use mean or median for categorical data?

Because categorical variables are non-numeric and have no numerical ordering.

What is mode imputation?

Replacing missing values with the most frequently occurring category.

When should mode imputation not be used?

When missing values are high or no category dominates.

What is Missing Category Imputation?

Creating a new category (e.g., "Missing") for all missing values.

Why can Missing Category Imputation be useful?

It preserves the information that the value was missing, which can itself be predictive.

Key Takeaways

1

Categorical variables should not use mean or median imputation.

2

Mode Imputation is the standard approach for low levels of missing categorical data.

3

A dominant category and approximately MCAR missingness make mode imputation effective.

4

Mode imputation changes category frequencies and can distort distributions if overused.

5

Missing Category Imputation is preferred when missing values are numerous or informative.

6

Always inspect the percentage of missing values, category distribution, and target distribution before selecting an imputation strategy.

7

SimpleImputer(strategy='most_frequent') implements mode imputation, while SimpleImputer(strategy='constant', fill_value='Missing') implements missing category imputation.

8

There is no universal best method — the right technique depends on the dataset, the missingness mechanism, and empirical validation through model performance.
