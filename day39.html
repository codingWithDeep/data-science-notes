What is KNN Imputer?

KNN stands for K-Nearest Neighbors.

It fills a missing value by finding rows that are most similar to the row containing the missing value.

Instead of calculating a mean or median, it asks: "Which rows look most similar to this row?" Then it uses those similar rows to estimate the missing value.

How Does KNN Measure Similarity?

Similarity is measured using Distance. Usually:

Euclidean Distance

Rows that are closer in distance are considered more similar.

Understanding Rows as Points

Imagine every row is a point in space.

3 features = 3D

4 features = 4D

5 features = 5D

Although we cannot visualize dimensions beyond three, mathematically it works exactly the same.

Euclidean Distance Formula

For two points (x₁, y₁) and (x₂, y₂):

d = √(x₂−x₁)² + (y₂−y₁)²

For three dimensions:

d = √(x₂−x₁)² + (y₂−y₁)² + (z₂−z₁)²

For n dimensions, we continue adding squared differences.

Problem with Missing Values

Normal Euclidean Distance cannot handle missing values.

ABC
102030
NaN2035

Difference for the first feature cannot be calculated.

Therefore, ordinary Euclidean Distance fails.

NaN Euclidean Distance

Scikit-Learn uses a modified version of Euclidean Distance specifically designed for missing values.

Instead of using every feature, it only compares dimensions where both rows have values. Missing dimensions are ignored.

Formula (Concept)

Distance = √( Weight × Σ(squared differences) )

Where:

Weight = Total features / Common (non-missing) features

Why is Weight Needed?

Suppose we have 4 features but only 2 are available.

Total = 4 / Common = 2 = 2

The distance is multiplied by this weight to compensate for ignored dimensions.

Suppose every feature exists.

Weight = 3/3 = 1

What Does "K" Mean?

K represents the number of nearest neighbors used.

K=1

Use only the nearest row

K=2

Use two nearest rows

(33 + 30) / 2 = 31.5
K=3

Use three nearest rows

(33+30+35) / 3

Complete KNN Imputation Algorithm

1

Identify missing value

2

Calculate distance from that row to every other row

3

Ignore missing dimensions while calculating distance

4

Use NaN Euclidean Distance

5

Find K nearest neighbors

6

Take the corresponding feature values

7

Average them (or use weighted average)

Advantages of KNN Imputer

Better Accuracy

Usually performs better than Mean, Median, and Mode imputation because it uses information from multiple features.

Preserves Relationships

Mean imputation destroys relationships between variables. KNN preserves feature interactions.

Data-Driven

Instead of inserting one fixed number, it uses actual nearby observations.

Works Well for Small/Medium Datasets

Especially useful when the dataset is not extremely large.

Disadvantages

1

Computationally Expensive

For every missing value, distance to every row must be calculated. Complexity increases rapidly with dataset size.

2

Slow

Large datasets require many calculations. Training becomes slow.

3

High Memory Usage

During prediction, the entire training dataset must be stored. Unlike Mean Imputation, KNN cannot simply remember one average.

4

Production Deployment

Incoming samples must be compared with the entire training data. Therefore:

More RAM More CPU Slower inference

Uniform vs Distance Weighting

This is an important concept.

Suppose two nearest neighbors are:

Neighbor 1

Distance: 10

Value: 70

Neighbor 2

Distance: 50

Value: 50

Uniform Weighting

Both neighbors contribute equally.

(70 + 50) / 2 = 60

Distance Weighting

Closer neighbors receive higher importance.

Weights: w = 1/distance

(70/10 + 50/50) / (1/10 + 1/50)

Value 70 influences more because its neighbor is much closer.

Observation from the Lecture

Changing to weights='distance' slightly increased accuracy compared to default 'uniform'. Not guaranteed for every dataset, but worth testing.

Comparison with Mean Imputation

The instructor compared Mean, Median, and KNN Imputation on the Titanic dataset.

KNN produced better performance.

Reason: It leverages relationships between multiple features instead of relying on a single statistic.

Best Practices

Standardize/normalize numerical features before KNN, because distance calculations are sensitive to feature scales.

Use KNN mainly for small to medium-sized datasets; for very large datasets it can become slow.

Tune n_neighbors using validation or cross-validation.

Compare weights='uniform' and weights='distance'.

Keep the default metric='nan_euclidean' unless you have a specific reason to change it.

Compare KNN with simpler imputers (Mean, Median) and more advanced methods (Iterative Imputer) to choose the best approach for your dataset.

Interview Questions

What is KNN Imputer?

A multivariate imputation technique that fills missing values using the values of the K most similar observations.

Why is it called multivariate?

Because it uses information from multiple features to estimate missing values.

Which distance metric is used by default?

NaN Euclidean Distance.

Why not use ordinary Euclidean Distance?

Because it cannot handle missing values; NaN Euclidean Distance ignores missing dimensions and adjusts the distance with a weighting factor.

What is the role of K?

It specifies how many nearest neighbors are used to estimate the missing value.

What is the difference between uniform and distance weights?

Uniform: Every neighbor contributes equally. Distance: Closer neighbors receive greater influence.

What are the major disadvantages of KNN Imputer?

High computational cost · Slow on large datasets · High memory usage · Requires the training data during inference/deployment

Key Takeaways

1

KNN Imputer is a multivariate missing value imputation technique — it uses information from multiple features, not just one.

2

It estimates missing values using similar rows rather than a single statistic (mean/median).

3

Similarity is measured using NaN Euclidean Distance — a modified distance that ignores missing dimensions and uses a weighting factor to compensate.

4

n_neighbors controls how many neighbors are used. weights='distance' often improves results because closer neighbors influence the prediction more.

5

Compared with Mean/Median imputation, KNN usually produces better predictive performance by preserving relationships between features.

6

The trade-off is increased computation time, memory usage, and deployment complexity (must store entire training data).

7

Always standardize features before KNN (distance is scale-sensitive), tune n_neighbors, and compare with other imputers to choose the best approach.
