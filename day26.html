3. Types of Categorical Data

Categorical data is mainly divided into two types:

Categorical Data │ ├── Nominal Data │ └── Ordinal Data

4. Nominal Categorical Data

Nominal data contains categories that do not have any meaningful order, ranking, or hierarchy.

Example: State

Assam West Bengal Karnataka Maharashtra

We cannot say Assam > Karnataka or Maharashtra < West Bengal. There is no ranking between the states. Therefore, State is a Nominal Categorical Variable.

Example: Engineering Branch

Computer Science Mechanical Civil Electrical Electronics

We cannot say Computer Science > Mechanical. These are only different categories. They do not have a natural order. Therefore, Engineering Branch is nominal data.

Other examples

  • Gender: Male, Female
  • Colors: Red, Green, Blue
  • City: Guwahati, Delhi, Mumbai
  • Blood group: A+, B+, AB+, O+

All these categories have no natural order.

5. Ordinal Categorical Data

Ordinal data contains categories that have a meaningful order, ranking, or hierarchy.

Example: Product Review

Poor < Average < Good

Here: Good > Average > Poor. Therefore, the categories have an order. So, Review is an Ordinal Categorical Variable.

More examples

  • Customer Satisfaction: Very Bad < Bad < Average < Good < Excellent
  • Education: High School < Undergraduate < Postgraduate
  • Size: Small < Medium < Large
  • Level: Low < Medium < High
  • Experience: Beginner < Intermediate < Advanced
  • Designation: Junior < Mid-Level < Senior

6. Why Do We Need Encoding?

Most categorical data is stored as text. However, most Machine Learning algorithms work with numbers and cannot directly understand strings such as "Good", "Poor", "Male", "Female".

Therefore, categorical values need to be converted into numerical values. This process is called:

Encoding
Converting categorical values into numerical values so that Machine Learning algorithms can process them.

7. Common Encoding Techniques

Encoding TechniqueUsed for

One-Hot EncodingNominal input features
Label EncodingTarget/output labels

8. Ordinal Encoding

Ordinal Encoding converts ordered categorical values into numerical values while preserving their order.

Suppose we have an education column:

Education
High School
Undergraduate
Postgraduate

The order is: High School < Undergraduate < Postgraduate

After Ordinal Encoding:

EducationEncoded Value
High School0
Undergraduate1
Postgraduate2

9. How Does Ordinal Encoding Work?

Suppose the feature is:

Education ----------- Postgraduate High School Undergraduate Postgraduate

First, we identify that education has a natural order. Then, we provide this order to OrdinalEncoder. The encoder converts High School → 0, Undergraduate → 1, Postgraduate → 2.

The transformed data becomes:

Education --------- 2 0 1 2
Important Point ⚠️
The Machine Learning model does not understand that Postgraduate > Undergraduate > High School. We must explicitly provide the correct order to OrdinalEncoder. The encoder cannot understand the semantic meaning of categories automatically.

13. Ordinal Encoder vs Label Encoder

This is the most important difference. ⚠️

OrdinalEncoder

OrdinalEncoder should be used for categorical input features (X).

  • Examples: Review, Education, Size, Experience Level
  • Can encode multiple feature columns at once.
  • Category order can be manually specified.

LabelEncoder

LabelEncoder should be used for the target/output variable (y).

  • Examples: Purchased (Yes/No), Placement (Placed/Not Placed), Rain (Yes/No), Email (Spam/Not Spam)
  • Usually encodes one-dimensional target values.
  • Category order is automatically decided.
OrdinalEncoderLabelEncoder
Used for input featuresUsed for target labels
Applied to XApplied to y
Can encode multiple columnsUsually encodes 1D target values
Order can be manually specifiedOrder is automatically decided

Remember: OrdinalEncoder → X   |   LabelEncoder → y

14. Train-Test Split

Before performing encoding, divide the dataset into training and testing data.

from sklearn.model_selection import train_test_split

# Select the input features and target
X = df.iloc[:, 2:4]
y = df.iloc[:, -1]

# Now perform the train-test split
X_train, X_test, y_train, y_test = train_test_split(
    X,
    y,
    test_size=0.2,
    random_state=0
)

Meaning: 80% → Training Data, 20% → Testing Data

15. Why Should We Split Before Encoding?

The correct process is:

Original Data ↓ Train-Test Split ↓ Fit Encoder on Training Data ↓ Transform Training Data ↓ Transform Testing Data

We should learn the transformation only from the training data. Otherwise, information from the test data may enter the training process. This is called:

Data Leakage
Rule: First → Train-Test Split   |   Then → Encoding

16. Import OrdinalEncoder

from sklearn.preprocessing import OrdinalEncoder

oe = OrdinalEncoder(
    categories=[
        ['Poor', 'Average', 'Good'],
        ['School', 'Undergraduate', 'Postgraduate']
    ]
)

17. Understanding the categories Parameter

The first input column is Review. Its order is: Poor < Average < Good.

The second input column is Education. Its order is: School < Undergraduate < Postgraduate.

Important Rule: The order of lists must match the order of columns in X_train. The first list belongs to Review, the second list belongs to Education.

18 & 19. Fit and Transform

Fit the encoder only on training data, then transform both:

oe.fit(X_train)

X_train_encoded = oe.transform(X_train)
X_test_encoded = oe.transform(X_test)

Shorter Method

X_train_encoded = oe.fit_transform(X_train)
X_test_encoded = oe.transform(X_test)
❌ Wrong:
X_test_encoded = oe.fit_transform(X_test)
Never fit the encoder separately on the test data.
✅ Correct:
Training Data → fit_transform()
Testing Data → transform()

20. Output of Ordinal Encoding

If a row contains Review = Good and Education = Undergraduate, the output will be [2, 1].

If a row contains Review = Average and Education = School, the output will be [1, 0].

21. Check Learned Categories

oe.categories_

Output:

[array(['Poor', 'Average', 'Good']), array(['School', 'Undergraduate', 'Postgraduate'])]

The underscore _ indicates an attribute that is created after fitting the encoder.

22 - 25. Label Encoding

Label Encoding is mainly used to convert categorical target labels into numerical labels.

from sklearn.preprocessing import LabelEncoder

le = LabelEncoder()

# Fit and transform
y_train_encoded = le.fit_transform(y_train)
y_test_encoded = le.transform(y_test)

Unlike OrdinalEncoder, we generally do not manually provide the category order. You can check learned classes using le.classes_.

26. Complete Code

import pandas as pd
from sklearn.model_selection import train_test_split
from sklearn.preprocessing import OrdinalEncoder, LabelEncoder

# Select input features and target variable
X = df[['Review', 'Education']]
y = df['Purchased']

# Train-test split
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.2, random_state=0)

# Create and apply OrdinalEncoder
oe = OrdinalEncoder(categories=[['Poor', 'Average', 'Good'], ['School', 'Undergraduate', 'Postgraduate']])
X_train_encoded = oe.fit_transform(X_train)
X_test_encoded = oe.transform(X_test)

# Create and apply LabelEncoder
le = LabelEncoder()
y_train_encoded = le.fit_transform(y_train)
y_test_encoded = le.transform(y_test)

28. Important Warning About LabelEncoder ⚠️

According to the Scikit-learn documentation: LabelEncoder should be used to encode target values y, not input features X.

❌ Avoid: le.fit_transform(X['Education']) for input feature encoding.
✅ Use: OrdinalEncoder() for ordinal input features.

29. Why Not Use LabelEncoder for Input Columns?

For ordinal features, we should explicitly define the correct order. LabelEncoder may assign values based on its internal class ordering (e.g., alphabetically) rather than the actual semantic ranking that we want.

30. What About Nominal Columns?

Suppose we have Gender (Male, Female). There is no meaningful order. We should not use Ordinal Encoding because it may create an artificial relationship (e.g., Male > Female).

Therefore, nominal input columns are generally encoded using One-Hot Encoding.

31. What If Different Columns Need Different Transformations?

Doing every transformation separately can become difficult. Scikit-learn provides ColumnTransformer.

ColumnTransformer allows us to apply different transformations to different columns in a single step. These transformations can later be combined using ColumnTransformer and Pipeline. (Will be studied in a later lesson).

32. Important Practical Notes ⭐

  1. Ordinal Encoding is used when categories have a meaningful order.
  2. One-Hot Encoding is generally used when categories have no order.
  3. Label Encoding is designed for target labels.
  4. Use OrdinalEncoder on input features (X).
  5. Use LabelEncoder on target values (y).
  6. Always perform the train-test split before fitting the encoder.
  7. Fit the encoder only on training data.
  8. Use the same fitted encoder to transform testing data.
  9. Never fit the encoder separately on testing data.
  10. For Ordinal Encoding, manually provide the correct category order.
  11. The order of category lists must match the order of input columns.
  12. Do not use Ordinal Encoding for nominal categories because it may create a false order.
  13. Do not use LabelEncoder for input feature columns.

Final Summary 🧠

Categorical Data │ ├── Nominal Data │ │ │ ├── No natural order │ │ │ ├── Examples: Gender, State, City, Branch │ │ │ └── One-Hot Encoding │ └── Ordinal Data │ ├── Natural order exists │ ├── Examples: Poor < Average < Good │ └── Ordinal Encoding

For the target:

Target Variable (y) │ ↓ Label Encoding

The easiest rule to remember is:

  • Ordinal Input Feature → OrdinalEncoder
  • Nominal Input Feature → OneHotEncoder
  • Categorical Target → LabelEncoder
