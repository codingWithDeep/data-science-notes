1. Introduction

Until now, we have studied many feature-engineering techniques, including: Handling missing values, Encoding categorical variables, One-Hot/Ordinal Encoding, ColumnTransformer, Mathematical transformations (FunctionTransformer, PowerTransformer), and Encoding numerical features.

In this lesson, we will study a slightly different feature-engineering problem:

How can we handle mixed data in a dataset?

Mixed-data columns and date-time columns are different from previous topics. They often require custom logic because there is no single universal transformation that works for every dataset.

2. What Is Mixed Data?

Mixed data occurs when a single column contains more than one type of information (e.g., Numerical and Categorical). Sometimes both types are present inside the same value, while in other cases they appear as separate values in the same column.

C85 │└── Numerical information: 85 └─── Categorical information: C

3. Why Is Mixed Data a Problem?

If we treat the complete values as categorical variables, every cabin may become a separate category (e.g., C85 → Category 1, C123 → Category 2). This may create many unique categories and increase dimensionality.

However, useful information is hidden inside these values. After separating the information:

  • The number of cabin categories becomes much smaller.
  • The numerical information is preserved.
  • The data becomes easier to analyse and encode.
  • Machine-learning models can use the information more effectively.

6. Analysing the Mixed `Number` Column

df["number"].unique()
# array([2, 3, 4, "Alone", 5])

df["number"].value_counts()

This confirms the column contains both Numbers and Text values.

7. Extracting Numerical Values

df["number_numerical"] = pd.to_numeric(
    df["number"],
    errors="coerce"
)
Explanation: pd.to_numeric() attempts to convert every value into a numerical value. Using errors="coerce" converts invalid numerical values (like "Alone") into NaN (Not a Number).

8. Creating the Categorical Column

df["number_categorical"] = np.where(
    df["number_numerical"].isnull(),
    df["number"],
    np.nan
)

Logic: If the numerical value is missing (isnull() == True), the original value was categorical, so copy it. Otherwise, store NaN.

9. Complete Code for Type 1 Mixed Data

import pandas as pd
import numpy as np

df["number_numerical"] = pd.to_numeric(df["number"], errors="coerce")

df["number_categorical"] = np.where(
    df["number_numerical"].isnull(),
    df["number"],
    np.nan
)

10 & 11. Handling the Titanic `Cabin` Column & Extracting Number

df["Cabin_num"] = df["Cabin"].str.extract(r"(\d+)")

df["Cabin_num"] = pd.to_numeric(df["Cabin_num"], errors="coerce")
Regular-expression explanation: r"(\d+)" means extract one or more consecutive digits. r (Raw string), () (Capturing group), \d (Any digit 0-9), + (One or more).

12. Extracting the Cabin Category

df["Cabin_cat"] = df["Cabin"].str[0]

# Alternative method:
df["Cabin_cat"] = df["Cabin"].str.extract(r"([A-Za-z]+)")

This extracts the first character (or all alphabetic characters) from every cabin value.

13. Complete Cabin Feature-Engineering Code

df["Cabin_num"] = pd.to_numeric(
    df["Cabin"].str.extract(r"(\d+)")[0],
    errors="coerce"
)

df["Cabin_cat"] = df["Cabin"].str[0]

14. Handling Missing Cabin Values

# For numerical cabin number:
df["Cabin_num"] = df["Cabin_num"].fillna(0) 
# (Or use median imputation in a Pipeline)

# For categorical cabin feature:
df["Cabin_cat"] = df["Cabin_cat"].fillna("Missing")

15. Reduction in Cardinality

Before extraction, df["Cabin"].nunique() may return a massive number. After extracting only the cabin category, it reduces to just a few letters (A, B, C, D, E, F, G, Missing).

Reducing unnecessary cardinality can: Reduce dimensionality, Improve model efficiency, Reduce memory usage, Reduce overfitting, and Improve generalisation.

16. Visualising Cabin Categories

df["Cabin_cat"].value_counts().plot(kind="bar")

This allows us to determine which cabin category contains the most passengers and whether cabin categories may be related to survival.

17, 18 & 19. Handling the Ticket Column

Ticket values like A/5 21171 or PC 17599 contain a categorical prefix and a numerical number.

Extracting Ticket Numbers:

df["Ticket_num"] = pd.to_numeric(
    df["Ticket"].str.extract(r"(\d+)$")[0],
    errors="coerce"
)

$ ensures it extracts numbers at the end of the string.

Extracting Ticket Categories:

df["Ticket_cat"] = (
    df["Ticket"]
    .str.replace(r"\d+", "", regex=True)
    .str.replace(r"[./]", "", regex=True)
    .str.strip()
)

df["Ticket_cat"] = df["Ticket_cat"].replace("", "Missing")
Why Is Ticket Extraction More Difficult? Cabin values usually have a simple format (Letter + Numbers). Ticket values may have many formats with spaces, periods, and slashes. There is no single regular expression that works perfectly for every dataset.
