Introduction

Introduction

In this lecture, the instructor continues the Feature Engineering series and starts discussing one of the most important preprocessing topics:

Outlier Detection and Removal

In the previous lecture, an introduction to outliers was covered. In this lecture, the focus is on the Z-Score Method, one of the most popular statistical techniques for detecting and handling outliers.

Concept

What is an Outlier?

An outlier is a data point that lies significantly far away from the majority of observations.

Example — Age Data

22 24 25 23 24 26 120 ← Outlier

The value 120 is extremely different from the remaining observations.

Outliers may occur because of:

Human data entry mistakes
Measurement errors
Equipment malfunction
Natural rare events
Genuine exceptional observations

Important: Not every outlier is wrong. Sometimes they carry valuable information. Always investigate before removing.

Core Technique

Z-Score Method

One of the most common techniques for detecting outliers is the Z-Score Method. However, this technique comes with a very important assumption.

Assumption of Z-Score

The feature (column) must be:

Normally Distributed

or

Approximately Normally Distributed

If the data is heavily skewed, the Z-score method should not be used.

Foundation

What is a Normal Distribution?

A Normal Distribution (Gaussian Distribution) is a bell-shaped curve.

Characteristics:

1 Most values are concentrated around the center (mean).
2 Fewer values exist as we move away from the center.
3 The distribution is symmetric.

Examples from real life:

Human Height

IQ Scores

Exam Marks

Blood Pressure

Measurement Errors

Empirical Rule

Properties of Normal Distribution

A normal distribution follows the famous 68–95–99.7 Rule.

±1σ

1 Standard Deviation

Range: μ − σ to μ + σ

68%

Contains approximately 68% of the observations.

±2σ

2 Standard Deviations

Range: μ ± 2σ

95%

Contains approximately 95% of the observations.

±3σ

3 Standard Deviations

Range: μ ± 3σ

99.7%

Contains approximately 99.7% of all observations.

Only 0.3% of observations lie outside this range. This is why observations beyond ±3σ are generally considered outliers.

Formula

Z-Score Formula

For every observation:

Z = X − μ σ
X

Data value (the observation)

μ

Mean of the feature

σ

Standard deviation

The Z-score tells us how many standard deviations a value is away from the mean.

Methods

Detecting Outliers

There are two equivalent approaches.

1

Boundary Method

Calculate upper and lower limits:

Upper Limit = μ + 3σ

Lower Limit = μ − 3σ

Any value:

< Lower Limit

or

> Upper Limit

→ is an outlier

2

Z-Score Method

Calculate Z-score for every observation.

−3 ≤ Z ≤ 3

→ Normal observation

Z < −3  or  Z > 3

→ Outlier

Both methods give the same result

Naming

Why is it called the Z-Score Method?

Because instead of directly checking Mean ± 3σ, we transform every observation into its standardized score (Z-score).

This transformation converts the feature into a standardized scale — making it easy to compare across different features with different units and scales.

Techniques

Handling Outliers

After detecting outliers, there are two major techniques.

1. Trimming

Also called Deletion or Removal

Simply remove all rows containing outliers.

Original dataset
Row Row Row Row Outlier Row Row
Remove outlier rows
Row Row Row Row Row Row

Original

1000

Outliers

5

995

Advantages

  • Very simple
  • Removes extreme noise

Disadvantages

  • Data loss
  • Dangerous if dataset is small
  • Genuine observations may be removed

2. Capping (Winsorization)

✦ Frequently preferred over trimming

Instead of deleting data, replace extreme values with the boundary values.

Upper Limit = 80

85 80 88 80 92 80

Lower Limit = 20

10 20 15 20 18 20

No rows are deleted.

Advantages

  • Preserves dataset size
  • Less information loss
  • Frequently preferred over trimming

5 Trimming in Code

trimming.py 
# Filter rows within bounds
new_df = df[(df['CGPA'] <= upper_limit) &
        (df['CGPA'] >= lower_limit)]

# Result: 5 rows removed
print(new_df.shape)  # (995, 3)

Result: Five rows removed. New dataset contains 995 rows instead of 1000 rows.

Alt Alternative Detection using Z-Score

zscore_detect.py 
# Calculate Z-score for every student
df['z_score'] = (df['CGPA'] - df['CGPA'].mean()) / df['CGPA'].std()

# Keep only -3 ≤ Z ≤ 3
new_df = df[(df['z_score'] >= -3) & (df['z_score'] <= 3)]

# The result is identical

6 Capping in Code

capping.py 
import numpy as np

# Replace values beyond upper limit with upper limit
# Replace values below lower limit with lower limit
df['CGPA'] = np.where(
    df['CGPA'] > upper_limit,
    upper_limit,
    np.where(
        df['CGPA'] < lower_limit,
        lower_limit,
        df['CGPA']
    )
)

Result After Capping

  • The number of rows remains unchanged.
  • Only the extreme values are adjusted.
  • Minimum value becomes 5.11
  • Maximum value becomes 8.80
  • The mean changes only slightly, while the influence of extreme values is greatly reduced.
Pros

Advantages of Z-Score Method

Easy to understand and implement.

Computationally efficient.

Works well on Gaussian (normal) data.

Widely used in statistics and machine learning.

Effective for detecting extreme observations.

Cons

Limitations of Z-Score Method

Assumes the data is normally distributed.

Mean and standard deviation are sensitive to outliers, so very extreme values can distort the thresholds.

Performs poorly on skewed or heavy-tailed distributions.

Not suitable for categorical variables.

Caution

When NOT to Use Z-Score

Avoid using it when:

Data is highly skewed.
Distribution is unknown and clearly non-normal.
The dataset contains many genuine extreme values.

In such cases, prefer methods like:

IQR (Interquartile Range)

For skewed distributions.

Modified Z-Score

Uses median and MAD — more robust.

Isolation Forest

For multivariate or complex datasets.

LOF / DBSCAN

Local Outlier Factor or DBSCAN for complex patterns.

Preparation

Interview Questions

Click each question to reveal the answer.

An outlier is a data point that lies significantly far away from the majority of observations. It can occur due to data entry mistakes, measurement errors, equipment malfunction, natural rare events, or genuine exceptional observations. Not every outlier is incorrect — some carry valuable information.
The Z-score method assumes that the feature (column) is normally distributed or approximately normally distributed. If the data is heavily skewed, this method should not be used as it can lead to incorrect outlier detection.
In a normal distribution: ±1σ contains ~68% of data, ±2σ contains ~95% of data, and ±3σ contains ~99.7% of data. Only 0.3% of observations lie outside ±3σ, which is why values beyond this range are considered outliers.
Z = (X − μ) / σ, where X is the data value, μ is the mean, and σ is the standard deviation. It tells us how many standard deviations a value is away from the mean.
According to the empirical rule, 99.7% of data in a normal distribution falls within ±3σ. Only 0.3% lies outside, making such observations extremely rare and statistically suspicious — hence they are flagged as outliers.
Trimming removes the outlier rows entirely, causing data loss. Capping (Winsorization) replaces extreme values with boundary values without deleting any rows, preserving dataset size. Capping is generally preferred.
The Z-score method relies on the symmetric properties of a normal distribution. In skewed data, the mean is pulled toward the tail, making the μ ± 3σ boundaries inaccurate. This can cause legitimate data points to be wrongly flagged as outliers, or actual outliers to be missed.
You can use visual methods like histograms, KDE plots, or Q-Q plots. Statistical tests like the Shapiro-Wilk test or Kolmogorov-Smirnov test can also be used. Skewness and kurtosis values close to 0 and 3 respectively also indicate normality.
Advantages: Preserves dataset size, less information loss, frequently preferred over trimming.
Disadvantages: Distorts the original distribution at the tails, may introduce artificial clustering at boundary values, and can mask the true nature of extreme observations.
I would choose the IQR (Interquartile Range) method because it uses the median and quartiles, which are robust to skewness. Unlike Z-score, it doesn't assume normality. The Modified Z-Score (using Median Absolute Deviation) is another excellent choice as it's also resistant to the influence of extreme values.
Final Notes

Key Takeaways

✔️

Z-score standardizes data by measuring how far each value is from the mean in terms of standard deviations.

✔️

It is reliable only for normally or approximately normally distributed features.

✔️

Values outside Mean ± 3 × Standard Deviation (or with |Z| > 3) are typically treated as outliers.

✔️

Outliers can be handled by Trimming (removing rows) or Capping/Winsorization (replacing extreme values with boundary values).

✔️

Always visualize the distribution (histogram, KDE plot, or Q-Q plot) before applying the Z-score method to verify the normality assumption.

✔️

In practice, IQR is often preferred for skewed data, while Z-score remains an excellent choice for approximately Gaussian features.
