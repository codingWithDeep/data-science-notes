Day 34 — Handling Date and Time Features 📅⏰

1. Introduction

In the previous lecture, we learned how to handle mixed variables. In this lesson, we will learn another important feature-engineering topic:

How to handle date- and time-based columns in a dataset.

Date and time columns often contain a large amount of hidden information. Instead of using the original date or time directly, we can extract useful features from it and provide those features to a machine-learning model. This lecture is mainly a code-based reference lesson covering useful pandas functions for date/time problems.

Part 1: Why Are Date and Time Columns Important?

Suppose a dataset contains the date: 22 August 2012. Although this looks like a single value, it contains many pieces of hidden information: Day of the month, Month, Year, Day of the week, Week number, Quarter, Semester, Whether it is a weekday or weekend, and Time elapsed from another date.

This process is known as date feature extraction or date feature engineering.

2. Real-World Example: Expense Tracker Application 💰

Suppose users record their expenses along with the date and time of each transaction. Date and time information can help answer questions such as:

  • Does the user spend more money on weekends or weekdays?
  • During which month or quarter is spending highest?
  • At what time of the day does the user usually spend money?
  • Are expenses higher during evenings or holidays?

3. Hidden Information Inside a Time Value

Suppose we have the time: 15:58:30. This single value contains Hour (15), Minute (58), and Second (30). Additional features can be created like Morning, Afternoon, Evening, Night, Business hours, and Time elapsed.

4. Datasets Used in the Lecture

Dataset 1: orders.csv (E-commerce order info: Date, Product, City, Quantity). Used for date-related feature engineering.

Dataset 2: messages.csv (Messaging app data: Date-time of message, Message content). Used for time-related feature engineering.

5. The Main Problem: Dates Are Often Imported as Strings

When a CSV file is imported, pandas may import the date column as object (strings). If it remains a string, we cannot directly extract Year, Month, Hour, etc.

The first task is to convert it into pandas datetime format.

6. Convert a Column into Datetime Format

df["date"] = pd.to_datetime(df["date"])

After conversion, the data type becomes datetime64[ns]. Now pandas datetime functions can be applied via the .dt accessor.

7 to 12. Extracting Basic Date Components

# 7. Year
df["year"] = df["date"].dt.year

# 8. Month Number (1-12)
df["month_number"] = df["date"].dt.month

# 9. Month Name (String)
df["month_name"] = df["date"].dt.month_name()

# 10. Day of the Month (1-31)
df["day"] = df["date"].dt.day

# 11. Day of the Week as Number (Monday=0 ... Sunday=6)
df["day_of_week"] = df["date"].dt.dayofweek

# 12. Day Name (String)
df["day_name"] = df["date"].dt.day_name()
Note on Day of Week: In pandas, numbering starts from Monday = 0 and ends with Sunday = 6.

13. Determine Whether a Date Is a Weekend

# Method 1: Using string names
df["is_weekend"] = (
    df["date"].dt.day_name()
    .isin(["Saturday", "Sunday"])
    .astype(int)
)

# Method 2: Shorter alternative using numbers (5=Sat, 6=Sun)
df["is_weekend"] = (df["date"].dt.dayofweek >= 5).astype(int)

import numpy as np

# Using quarters
df["semester"] = np.where(df["date"].dt.quarter.isin([1, 2]), 1, 2)

17 to 20. Calculating Time Differences ⏳

today = pd.Timestamp("2021-04-21")

# 17. Time Elapsed (Returns Timedelta object: days, hours, mins, secs)
df["time_elapsed"] = today - df["date"]

# 18. Extract Only the Number of Days
df["days_elapsed"] = (today - df["date"]).dt.days

# 19. Approximate Number of Months
df["months_elapsed"] = (today - df["date"]) / np.timedelta64(1, "D") / 30.44

# 20. Approximate Years
df["years_elapsed"] = (today - df["date"]) / np.timedelta64(1, "D") / 365.25
Note on Months/Years: Months do not have a fixed length (28, 29, 30, or 31 days). Therefore, month and year calculations are often approximate unless calendar-aware logic is used. 365.25 accounts for leap years.

Part 2: Handling Time Features ⏰

Now the lecture moves to the second dataset containing message timestamps like 2019-08-10 15:58:30.

21. Convert the Message Date Column into Datetime

time_df["date"] = pd.to_datetime(time_df["date"], errors="coerce")

22 to 25. Extracting Time Components

# 22. Extract Hour (0-23)
time_df["hour"] = time_df["date"].dt.hour

# 23. Extract Minute (0-59)
time_df["minute"] = time_df["date"].dt.minute

# 24. Extract Second (0-59)
time_df["second"] = time_df["date"].dt.second

# 25. Extract Only the Time Portion
time_df["time_only"] = time_df["date"].dt.time

26 to 30. Calculating Time Differences & Conversions

today = pd.Timestamp.now()

# 26. Calculate Time Differences
difference = today - time_df["date"]

# 27. Convert to Seconds
time_df["seconds_elapsed"] = difference.dt.total_seconds()

# 28. Convert to Minutes
time_df["minutes_elapsed"] = difference.dt.total_seconds() / 60

# 29. Convert to Hours
time_df["hours_elapsed"] = difference.dt.total_seconds() / 3600

# 30. Convert to Days
time_df["days_elapsed"] = difference.dt.total_seconds() / 86400
1 minute = 60 seconds 1 hour = 60 minutes = 3,600 seconds 1 day = 24 hours = 86,400 seconds

31. Additional Feature: Part of the Day 🌅🌙

def part_of_day(hour):
    if 5 <= hour < 12: return "Morning"
    elif 12 <= hour < 17: return "Afternoon"
    elif 17 <= hour < 21: return "Evening"
    else: return "Night"

time_df["part_of_day"] = time_df["hour"].apply(part_of_day)

33. Handling Invalid Date Values ⚠️

Real datasets may contain invalid values like 2024-13-50 or Unknown. Using standard pd.to_datetime() may raise an error. A safer method is:

df["date"] = pd.to_datetime(df["date"], errors="coerce")
Invalid dates are converted into NaT (Not a Time), which is the datetime equivalent of NaN.

34. Handling Different Date Formats

35. Time Zones 🌍

# Localize to a timezone
df["date"] = df["date"].dt.tz_localize("Asia/Kolkata")

# Convert to another timezone
df["new_york_time"] = df["date"].dt.tz_convert("America/New_York")
Timezone handling requires care because of UTC offsets and Daylight-saving time. It is relatively less common in beginner projects but important in Global apps, Server logs, and Financial markets.

36. Important Machine-Learning Consideration

You should not automatically extract every possible date feature. Create only features relevant to the problem using Domain knowledge and the business problem.
  • Expense prediction: Month, Weekend, Day of week, Part of day.
  • Fraud detection: Hour, Minute, Time since previous transaction.
  • Sales forecasting: Month, Quarter, Week number, Holiday indicator.

37. Important Issue: Cyclical Features 🔄

Date and time features are cyclical. E.g., December (12) and January (1) are adjacent months, but their numeric difference is 11. To fix this for advanced ML, encode using Sine and Cosine:

# For the month (12 months cycle)
df["month_sin"] = np.sin(2 * np.pi * df["month_number"] / 12)
df["month_cos"] = np.cos(2 * np.pi * df["month_number"] / 12)

# For the hour (24 hours cycle)
df["hour_sin"] = np.sin(2 * np.pi * df["hour"] / 24)
df["hour_cos"] = np.cos(2 * np.pi * df["hour"] / 24)

38. Complete Date Feature-Engineering Code

import pandas as pd
import numpy as np

df = pd.read_csv("orders.csv")

# Convert into datetime safely
df["date"] = pd.to_datetime(df["date"], errors="coerce")

# Basic date features
df["year"] = df["date"].dt.year
df["month_number"] = df["date"].dt.month
df["month_name"] = df["date"].dt.month_name()
df["day"] = df["date"].dt.day
df["day_of_week"] = df["date"].dt.dayofweek
df["day_name"] = df["date"].dt.day_name()

# Weekend feature
df["is_weekend"] = (df["date"].dt.dayofweek >= 5).astype(int)

# Week number (Modern pandas)
df["week_number"] = df["date"].dt.isocalendar().week

# Quarter & Semester
df["quarter"] = df["date"].dt.quarter
df["semester"] = np.where(df["date"].dt.month <= 6, 1, 2)

# Date difference
today = pd.Timestamp.now()
df["days_elapsed"] = (today - df["date"]).dt.days

print(df.head())

39. Complete Time Feature-Engineering Code

import pandas as pd

time_df = pd.read_csv("messages.csv")

# Convert into datetime safely
time_df["date"] = pd.to_datetime(time_df["date"], errors="coerce")

# Extract time components
time_df["hour"] = time_df["date"].dt.hour
time_df["minute"] = time_df["date"].dt.minute
time_df["second"] = time_df["date"].dt.second
time_df["time_only"] = time_df["date"].dt.time

# Calculate elapsed time
today = pd.Timestamp.now()
difference = today - time_df["date"]

time_df["seconds_elapsed"] = difference.dt.total_seconds()
time_df["minutes_elapsed"] = difference.dt.total_seconds() / 60
time_df["hours_elapsed"] = difference.dt.total_seconds() / 3600

40. Final Summary 📌

The main steps for handling date and time features are:

  1. Import dataset: df = pd.read_csv("data.csv")
  2. Check data types: df.info()
  3. Convert object to datetime: df["date"] = pd.to_datetime(df["date"])
  4. Extract date features: .dt.year, .month, .month_name(), .day, .dayofweek, .day_name(), .isocalendar().week, .quarter
  5. Create custom features: Weekend indicator, Semester, Part of the day, Time elapsed.
  6. Extract time components: .hour, .minute, .second, .time
  7. Calculate differences: today - df["date"] and convert to Days/Hours/Minutes/Seconds.

Main Takeaway 🎯

A date or timestamp is not merely one column. It can contain many hidden and useful features. For example:

2025-08-10 15:58:30 │ ├── Year → 2025 ├── Month → 8 ├── Month name → August ├── Day → 10 ├── Day name → Sunday ├── Weekend → Yes ├── Week number → 32 ├── Quarter → 3 ├── Semester → 2 ├── Hour → 15 ├── Minute → 58 ├── Second → 30 └── Part of day → Afternoon

These extracted features can significantly improve machine-learning models when they are relevant to the problem.
