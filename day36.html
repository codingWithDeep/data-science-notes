Introduction

In the previous lecture, we learned that if a dataset contains missing values, one option is to delete the rows containing missing data. However, deleting data is not always a good idea because it reduces the dataset size and may remove useful information.

In this lecture, we begin studying how to fill (impute) missing values instead of deleting them.

Important Scope

Today's lecture is completely focused on Numerical Features. Categorical missing data will be discussed separately.

1

Mean / Median Imputation

This is the simplest and one of the most commonly used methods.

Suppose we have:

Age
22
25
NaN
30
35

Compute the Mean or Median, then replace every missing value with that number.

Mean vs Median

Use Mean when

The data is approximately Normally Distributed.

In a normal distribution:

Mean ≈ Median ≈ Mode

Therefore using Mean is acceptable.

Use Median when

The distribution is Skewed.

Since Mean is pulled toward outliers:

Median becomes a better representative.

Advantages of Mean/Median Imputation

Very Simple

  • Easy to understand
  • Easy to implement
  • Only one line of code

Easy in Production

Deploying ML models becomes easier because only one statistic needs to be stored.

# Store
Age Median = 28

# Whenever new user has missing Age,
# replace with 28

Works Well When Missing Values Are Few

Generally works reasonably well if:

Missing < 5%

This is an important rule.

Disadvantages

Although simple, it changes important statistical properties.

1

Changes Distribution

Original

After Mean

Many observations pile up around Mean. Distribution shape changes.

2

Reduces Variance

Variance measures spread. Since many observations become identical, spread decreases.

Variance decreases
3

Alters Covariance / Correlation

Suppose Age is strongly related to Salary. Replacing Age with a constant breaks this relationship.

Correlation changes

Covariance changes

4

Affects Outliers

Because IQR changes, boxplot outliers may increase or decrease.

When Should Mean/Median Be Used?

Use when both conditions are met:

Missing Completely At Random (MCAR)

No pattern in missingness

Missing percentage is small

Preferably below 5%

Although people use Mean Imputation frequently, there are more sophisticated methods that often perform better.

Practical Example — Titanic Dataset

Features

Age Fare Family Survived

Missing Values Setup

  • Age already had missing values
  • Fare — artificially introduced 5% missing values

Goal

Fill Age and Fare using Mean and Median.

Train-Test Split — Critical Rule

Always split data first. Never calculate Mean or Median using the entire dataset.

✓ Correct Process

Train
Calculate Mean
Store Mean
Apply to Train & Test

✗ Wrong Process

Never calculate Mean separately on Test data.

⚠ Data Leakage

Calculating statistics on test data causes Data Leakage — this is a very important interview question.

Coding — Without Scikit-Learn

Calculate the statistics, then use fillna():

# Calculate
age_median = df['Age'].median()
age_mean = df['Age'].mean()
fare_median = df['Fare'].median()
fare_mean = df['Fare'].mean()

# Fill missing values
df['Age_median'] = df['Age'].fillna(age_median)
df['Age_mean'] = df['Age'].fillna(age_mean)
df['Fare_median'] = df['Fare'].fillna(fare_median)
df['Fare_mean'] = df['Fare'].fillna(fare_mean)

Create separate columns to compare results: Age_original, Age_mean, Age_median, etc.

  • Very little change in all checks

Age — Not Recommended

from sklearn.compose import ColumnTransformer

transformer = ColumnTransformer(
  transformers=[
    ('age_imputer', SimpleImputer(strategy='median'), ['Age']),
    ('fare_imputer', SimpleImputer(strategy='mean'), ['Fare'])
  ]
)

Everything is transformed together.

Important Rule — Preventing Data Leakage

✓ Always Do This

1 Fit on Training Data
2 Transform Training Data
3 Transform Test Data

✗ Never Do This

Fit on Test Data

This causes Data Leakage — your model sees information from the test set during training.

2

Arbitrary Value Imputation

Instead of Mean or Median, replace every missing value with a fixed number.

-1 999 10000 -999

The chosen value should ideally not already exist in the data.

Why Use It?

"The goal is to tell the Machine Learning model: This observation originally had missing data."

The model may learn useful patterns from the missingness itself.

Advantages

Very easy
Simple
Works for numerical & categorical

Disadvantages

Same issues as Mean Imputation, plus:

  • Distribution changes
  • Variance changes (sometimes increases due to extreme values)
  • Correlation changes

Practical Observation

Age Variance (Original)

204

After replacing with 999

Much Larger ↑

Distribution developed a new spike.

When Should It Be Used?

When data is Not Missing At Random (NMAR)

Example: People with very high income intentionally do not reveal salary. Missingness itself contains information. Instead of hiding it, preserve it by using an arbitrary value.

3

End of Distribution Imputation

This is an improvement over Arbitrary Value Imputation. Instead of choosing 999 or -999, choose a value from the extreme end of the distribution.

Normal Distribution

Mean + 3 × Std Deviation

or

Mean − 3 × Std Deviation

Values lie beyond almost all observations (extreme tail/outlier region)

Skewed Distribution

Upper: Q3 + 1.5 × IQR

Lower: Q1 − 1.5 × IQR

where IQR = Q3 − Q1

Why This Method?

Instead of inventing 999, choose an extreme value based on the actual distribution. This keeps the replacement more realistic while still signaling that the value was originally missing.

Advantages

  • Easy to implement
  • Uses statistics derived from the data rather than completely arbitrary constants
  • Preserves the "missingness signal" by placing imputed values at the distribution's edge

Disadvantages

  • Distribution changes
  • Variance may increase
  • Correlation/Covariance changes
  • Introduces artificial extreme values (outliers)

When Should It Be Used?

Primarily when the data is Not Missing At Random (NMAR) and you want the model to recognize that the value was missing.

4

Random Sample Imputation

Next Lecture

The instructor briefly introduces this method but covers it in the next video.

Core Idea

Randomly select an existing value from the same feature and use it to replace each missing value.

This often preserves the original distribution better than Mean/Median imputation.

Choosing the Best Imputation Technique

There is no single best method for every dataset.

Recommended Workflow

1

Try multiple imputation methods

2

Compare their impact on:

Variance Distribution Correlation/Covariance Outliers (Boxplots)
3

Evaluate model performance using cross-validation

4

Select the technique that produces the best predictive performance while preserving statistical properties

Key Takeaways

1

Missing values can be handled either by deleting rows or by imputing values.

2

Imputation is divided into Univariate (same feature) and Multivariate (other features) approaches.

3

Mean works best for approximately normal distributions; Median is preferred for skewed distributions.

4

Mean/Median imputation is suitable when data is Missing Completely At Random (MCAR) and the missing percentage is low (typically <5%).

5

Mean/Median imputation changes the distribution, reduces variance, alters correlations, and may affect outlier detection.

6

Always evaluate variance, distribution plots, correlation/covariance, and boxplots after imputation.

7

Fit imputers only on the training data, then transform both training and test data to avoid data leakage.

8

SimpleImputer supports mean, median, most_frequent, and constant strategies.

9

ColumnTransformer allows different imputation strategies for different columns.

10

Arbitrary Value Imputation uses a fixed value (e.g., -1 or 999) to preserve the information that data was missing, especially useful when missingness is informative (NMAR).

11

End of Distribution Imputation replaces missing values with statistically extreme values (e.g., Mean ± 3σ or Q3 + 1.5×IQR), providing a more data-driven alternative to arbitrary constants.

12

Random Sample Imputation and automatic technique selection are covered in the following lecture.
