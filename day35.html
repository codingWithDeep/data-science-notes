Day 35 — Handling Missing Data: Complete Case Analysis (CCA)

Feature Engineering (Complete Notes for Data Science Students)

Introduction

In this lecture, we begin one of the most important topics in Feature Engineering:

Handling Missing Data (Missing Values)

Almost every real-world dataset contains missing values. Before training any machine learning model, a data scientist must decide how to deal with them.

Why is Handling Missing Data Important?

Most machine learning algorithms cannot handle missing values (NaN). If you train a model using data containing missing values, many algorithms will throw an error, some models refuse to train, and others may produce poor results.

Handling missing values is one of the primary responsibilities of a Data Scientist before model training.

Two Ways to Handle Missing Data

Method 1: Remove

Instead of filling missing values, simply remove the rows (or columns) containing missing values. (Today's topic)

Method 2: Impute

Instead of deleting rows, replace missing values with meaningful values. (Covered in next lectures)

Types of Imputation

1. Univariate Imputation

Handle one column independently. Different techniques are used depending on the data type:

  • Numerical: Mean, Median, Random Value, End of Distribution Imputation.
  • Categorical: Mode (Most Frequent Value), "Missing".
Univariate Imputation │ ├── Numerical Columns │ ├── Mean Imputation │ ├── Median Imputation │ ├── Random Value Imputation │ └── End of Distribution Imputation │ └── Categorical Columns ├── Mode (Most Frequent Value) └── Replace with "Missing"

2. Multivariate Imputation

Multiple columns are used together to predict the missing value using other features. E.g., KNN Imputer and Iterative Imputer.

Today's Topic: Complete Case Analysis (CCA)

Definition of Complete Case Analysis

Complete Case Analysis means: Remove every row that contains at least one missing value.

Example:

AgeSalaryExperienceTarget
25500003Yes
NaN400002Yes
40700008No

The second row contains a missing value. After CCA, the entire second row is removed.

Why is it called Complete Case Analysis? After deleting incomplete rows, only rows with complete information remain. The dataset contains only "complete cases".

The Assumption Behind CCA: MCAR

Crucial Assumption ⚠️

CCA should only be applied when data is Missing Completely At Random (MCAR). Missingness is purely random; it does not depend on any feature, target, or hidden variable. If missingness has a pattern (e.g., only females missing), CCA is wrong to use because it introduces bias.

Advantages of Complete Case Analysis

Pros

  • Very Easy: Just one line of code: df.dropna().
  • No Data Manipulation: No need to calculate mean, median, or mode.
    • Distribution Remains Similar: Statistical distribution stays almost unchanged if missingness is truly random.

Cons

  • Data Loss: Biggest drawback. Deleting rows means losing valuable information.
    • Distribution May Change: If missingness is not MCAR, removing rows changes the distribution and makes the model biased. Production Problem: If trained on clean data, the model fails when new missing data arrives.
When Should You Use CCA?
Condition 1: Data must be Missing Completely At Random (MCAR).
Condition 2: Missing percentage should be less than ~5%.
High Missing %: (e.g., 90-98%) ➡ Remove the column itself.

What if Missing Percentage is Very High?

Rule of Thumb:
Small missing percentage ➡ Remove rows.
Huge missing percentage ➡ Remove the column itself.

Selecting Columns Automatically

Instead of checking manually, identify low-missing columns automatically:

cols = [col for col in df.columns if df[col].isnull().mean() < 0.05]

Removing Rows

new_df = df.dropna(subset=cols)

This removes rows having missing values only in the selected low-missing columns.

Checking Data Loss

print(f"Data Retained: {len(new_df) / len(df) * 100:.2f}%")

If around 94% of data is retained, CCA is acceptable.

import matplotlib.pyplot as plt

Best Practices Before Applying CCA:
  • ✅ Is the missingness likely MCAR?
  • ✅ Is the missing percentage <5%?
  • ✅ How many rows will remain?
  • ✅ Do numerical distributions remain similar?
  • ✅ Do categorical proportions remain similar?

Key Takeaways

  • Most ML algorithms cannot directly handle missing values.
  • Two strategies: Remove missing data or Impute it.
    • CCA (Complete Case Analysis) removes every row with at least one missing value in selected columns. Use CCA only when: Missingness is MCAR AND percentage < 5%. Advantages: Simple, fast, preserves distribution if MCAR. Disadvantages: Huge data loss, bias if not MCAR, breaks in production. High missing %: Drop the column instead of rows. Always verify histograms/KDE (numerical) and category proportions (categorical) before and after CCA. In practice: Imputation is preferred over CCA for robust production models.
