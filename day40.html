Introduction

In the previous few lectures, we learned different techniques for handling missing values.

So far we have covered:

Mean/Median Random Sample Missing Indicator KNN Imputer Other basic techniques

Today we study one of the most important and most widely used multivariate imputation techniques:

MICE

Also called: Iterative Imputer (Scikit-Learn) · Multiple Imputation by Chained Equations

Although the algorithm may initially seem difficult because it contains multiple iterative steps, once you understand the workflow, it becomes quite logical.

?

What is MICE?

MICE stands for Multivariate Imputation by Chained Equations.

Instead of filling missing values using a simple statistic like mean or median, MICE predicts missing values using relationships between all other variables.

Instead of: "Replace Age with mean."

It says: "Predict Age using Salary, Education, Experience, etc."

This makes the imputed values much more realistic.

Why is it Called "Chained Equations"?

Suppose your dataset has all three columns containing missing values:

Age Salary Experience

MICE performs prediction in a chain:

1. Predict missing Age
2. Predict missing Salary
3. Predict missing Experience
4. Repeat again
5. Repeat until stable

Since every column depends on predictions from previous columns, the equations form a chain. Hence the name.

Types of Missing Data

Before applying MICE, we must understand the type of missingness. There are three categories.

1

MCAR — Missing Completely At Random

The missing values occur completely randomly. There is no relationship between missingness and any other variable.

Example: A hospital database loses data because the server crashed for one day. Patients' blood pressure records are missing.

This missingness has nothing to do with patient age, disease, or income. It is completely random.

Power failure Server crash Sensor failure Random corruption
Random missingness No hidden pattern Hard to predict
2

MAR — Missing At Random

Most Important for MICE

The missing value depends on other observed variables, not on the missing variable itself.

A survey asks Age, Salary, Education. Some people skip Salary. But maybe older people or highly educated people skip salary.

Salary is missing, but Age and Education help predict Salary.

AgeEducationSalary
25Graduate50000
30MastersNaN
40PhD80000

Salary can be predicted using Age and Education.

3

MNAR — Missing Not At Random

Missingness depends on the missing value itself.

Rich people intentionally do not disclose income.

Low-performing students hide their grades.

Other columns may not explain the missing values well. MICE performs poorly here.

Best Case: Missing At Random (MAR)

Because other variables contain enough information to predict missing values. It can still be applied elsewhere, but results are generally best with MAR.

Advantages of MICE

Produces Natural Values

Unlike mean imputation, predicted values look realistic.

Uses Relationships Between Features

Instead of treating every feature independently, it learns correlations.

High Accuracy

Generally produces much better imputations than Mean, Median, Mode, and Random Sampling.

Preserves Data Distribution

Since predictions are model-based, the original feature distribution is better maintained.

Disadvantages

1

Slow

Many machine learning models are trained repeatedly. So it takes longer than simple imputation.

2

High Memory Usage

The complete training data must remain available. Future missing values are imputed using the trained models. Large datasets require more RAM.

3

Computationally Expensive

Every iteration retrains models. Many iterations increase computation time.

Why predict again?

During Iteration 1, other missing values were still represented by mean values — poor approximations. Predictions made using them are not optimal. So we repeat. Each iteration improves previous predictions.

Iteration 1 — uses poor estimates
Iteration 2 — uses better estimates

Eventually — predictions stabilize

ML Models That Can Be Used

MICE does not require a single fixed algorithm. You can choose different estimators:

Linear Regression Bayesian Ridge (default) Decision Tree Random Forest Extra Trees Gradient Boosting XGBoost LightGBM CatBoost KNN Regressor Logistic Regression (categorical) Random Forest Classifier (categorical)

Time Complexity

O( Iterations × Features × Model Training Cost )

Therefore, more iterations mean higher computation time.

Practical Tips

Standardize/scale features when the estimator benefits from it (e.g., Bayesian Ridge, Linear Regression).

Encode categorical variables before using MICE (or use models that handle them appropriately).

MICE works best when features are correlated. If two variables have almost no relationship, prediction quality will be poor.

Do not use the target variable while imputing features to avoid data leakage unless you have a carefully justified supervised imputation strategy.

Fit the imputer only on the training set, then transform both training and test sets using the fitted imputer.

Scikit-Learn Implementation

from sklearn.experimental import enable_iterative_imputer
from sklearn.impute import IterativeImputer

imputer = IterativeImputer(
  max_iter=10,
  random_state=42
)

X_imputed = imputer.fit_transform(X)

Important Parameters

max_iter

Number of MICE iterations.

estimator

Regression model used for prediction (default: Bayesian Ridge).

random_state

Ensures reproducible results.

initial_strategy

Initial filling method ('mean', 'median', etc.).

When to Use MICE

  • Missing values are MAR
  • Numerical features with meaningful correlations
  • Moderate missingness (~30–40%)
  • Accuracy is more important than speed

When to Avoid MICE

  • Very large datasets where time is critical
  • Features with almost no relationship
  • MNAR situations
  • Real-time systems requiring fast inference

Interview Questions

What is MICE?

MICE (Multivariate Imputation by Chained Equations) is an iterative, model-based imputation technique that predicts missing values using other features in the dataset.

Why is MICE better than Mean Imputation?

Because it preserves relationships between variables and generates more realistic values instead of replacing everything with the same statistic.

Why are multiple iterations required?

Early predictions rely on rough initial estimates (often means). Repeating the process improves these estimates until the predictions stabilize (convergence).

Which missing data mechanism is most suitable for MICE?

Missing At Random (MAR).

Why is MICE computationally expensive?

Because it repeatedly trains machine learning models for each feature with missing values across multiple iterations.

Key Takeaways

1

MICE = Multivariate Imputation by Chained Equations. Also known as Iterative Imputation in Scikit-Learn.

2

It predicts missing values using other features, rather than simple statistics.

3

Best suited for MAR (Missing At Random) data.

4

Starts with an initial fill (usually mean), then iteratively refines predictions.

5

Each feature with missing values is modeled separately while using the remaining features as predictors.

6

The process repeats until predictions converge or the maximum number of iterations is reached.

7

Produces more accurate and natural imputations than simple methods, but at the cost of increased computation time and memory usage.

8

It is widely used in real-world machine learning and statistical modeling when preserving feature relationships is important.

9

Default estimator is Bayesian Ridge Regression, but you can use any regressor (Random Forest, XGBoost, etc.).
