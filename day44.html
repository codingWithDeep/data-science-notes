Definition

What is the Percentile Method?

The Percentile Method identifies outliers by selecting a percentage of observations from both ends (lower and upper tails) of the distribution.

Instead of using standard deviations (Z-score) or quartiles (IQR), this method uses percentiles.

Very simple

Easy to understand

Works for many datasets

Flexible thresholds

Foundation

First Understand Percentiles

A percentile tells us how many observations lie below a particular value.

Suppose you scored in the 98th percentile in an exam. It means you scored better than 98% of students. Only 2% scored higher than you.

Two Options

Two Things You Can Do

Once outliers are detected, there are only two options.

Option 1: Trimming

Removing Outliers

Delete every observation beyond the threshold.

If Height > 99th percentile → Delete

If Height < 1st percentile → Delete

Simple.

Option 2: Capping

✦ Often preferred

Instead of deleting, replace extreme values.

Suppose 95th percentile = 70

80 70 85 70 90 70 95 70

No data is deleted. Only extreme values are limited.

Named Technique

Winsorization

Capping using percentiles is called Winsorization — named after the statistician Charles P. Winsor.

Instead of removing extreme observations, they are replaced with the nearest acceptable values.

Original

2 5 7 8 9 11 200

After Winsorization

5 5 7 8 9 11 11

No observations are deleted. Only extremes are capped.

The instructor uses a dataset containing Gender, Height, Weight. Only Height is used because it clearly contains outliers.

Step-by-Step

1

Finding Upper Limit (99th Percentile)

upper_limit = df['Height'].quantile(0.99)
# ≈ 74.85
2

Finding Lower Limit (1st Percentile)

lower_limit = df['Height'].quantile(0.01)
# ≈ 63.50
3

Trimming Code

new_df = df[
    (df['Height'] >= lower_limit) &
    (df['Height'] <= upper_limit)
]

Implementation

Winsorization (Capping) Code

winsorize.py
import numpy as np

df['Height'] = np.where(
    df['Height'] >= upper_limit,
    upper_limit,
    np.where(
        df['Height'] <= lower_limit,
        lower_limit,
        df['Height']
    )
)

Alternative Readable Code

winsorize_readable.py
# Cap upper values
df['Height'] = np.where(
    df['Height'] > upper_limit,
    upper_limit,
    df['Height']
)

# Cap lower values
df['Height'] = np.where(
    df['Height'] < lower_limit,
    lower_limit,
    df['Height']
)

After Winsorization

  • Dataset size remains exactly the same.
  • Only extreme values change.
  • Maximum value decreases, Minimum value increases.
  • Mean changes slightly, Standard deviation decreases.
  • Count remains unchanged.

From the Lecturer

Important Practical Notes

1

There is no universally perfect percentile.

Try different values:

99% 99.5% 95% 97% 98%

Choose whichever gives the best practical result.

2

Higher Percentile Means Less Data Loss

99.9%

Very few observations removed

95%

Much more data removed

Higher thresholds preserve more data.

3

Different Libraries Exist

from scipy.stats.mstats import winsorize

winsorize(data, limits=[0.01, 0.01])

The instructor prefers simple NumPy code — easy to understand, avoids unnecessary dependencies.

4

Some Implementations Replace with Slightly Different Values

Some practitioners replace values with:

upper_limit + 1 lower_limit - 1

There is no strict rule — use the approach that produces the best results for your specific problem.

Guideline

When Should You Use It?

Use the Percentile Method when:

You have numerical features.
You want a simple and robust outlier handling technique.
The data is skewed or does not perfectly follow a normal distribution.
You prefer controlling how aggressive outlier removal should be.

Final Notes

Final Summary

The Percentile Method is a simple yet powerful approach for outlier detection. By defining lower and upper percentile thresholds (such as the 1st and 99th percentiles), you can either trim outliers by removing them or winsorize them by replacing extreme values with the nearest acceptable percentile boundary.

Unlike the Z-Score method, it does not require normally distributed data, and unlike the IQR method, it allows you to control exactly how much of the data is treated as extreme. This flexibility makes it a popular choice in real-world data preprocessing and feature engineering workflows.
