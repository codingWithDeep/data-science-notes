Introduction

Introduction

In the previous lesson (Day 42), we learned about Outlier Detection using the Z-Score Method, which is mainly used when the data follows a Normal (Gaussian) Distribution.

In this lecture, we learn the second technique for detecting and treating outliers:

Interquartile Range (IQR) Method

This technique is especially useful when the dataset is skewed (not normally distributed).

Concept

What are Outliers?

An outlier is a data point that is significantly different from the rest of the observations.

Example

10 12 15 18 20 22 24 100 ← Outlier

100 is an outlier because it is much larger than the other values.

Outliers may occur because of:

Human data entry mistakes
Measurement errors
Fraudulent activities
Genuine rare observations

Remember: Not every outlier should be removed.

When to Use

When Should You Use the IQR Method?

Use the IQR Method when your data is skewed and NOT normally distributed.

Examples:

Income

Salary

House Prices

Exam Marks

Medical Costs

These variables usually have a long tail.

Caution

When NOT to Use IQR

If the data follows a Normal Distribution, then the Z-score Method is generally preferred.

Definition

What is IQR?

IQR stands for Interquartile Range.

It measures the spread of the middle 50% of the data.

IQR = Q3 − Q1

Example

Q1

17

IQR = 44 − 17 = 27

Q3

44

The lecture's dataset had an IQR of approximately 27.

Visualization

Understanding the Box Plot

A Box Plot is a graphical representation of numerical data.

The box contains Q1, Median (Q2), and Q3. The whiskers extend to the limits. Anything beyond is an outlier.

Core Rule

IQR Proximity Rule

Upper Limit

Upper = Q3 + 1.5 × IQR

Lower Limit

Lower = Q1 − 1.5 × IQR

Any value greater than Upper Limit or less than Lower Limit is considered an outlier.

Deep Dive

Why Multiply by 1.5?

John Tukey introduced the factor 1.5 as a practical rule for identifying unusually distant observations.

1.5 × IQR

Detects mild outliers — used by default in box plots.

3 × IQR

Detects extreme outliers.

It works well across many skewed distributions without assuming normality.

1

Calculate Quartiles

Q1 = df['placement_exam_marks'].quantile(0.25)
Q3 = df['placement_exam_marks'].quantile(0.75)

# Q1 = 17, Q3 = 44
2

Calculate IQR

IQR = Q3 - Q1

# IQR = 27
3

Calculate Limits

lower_limit = Q1 - 1.5 * IQR
upper_limit = Q3 + 1.5 * IQR

# Lower = -23.5, Upper = 84.5

Since exam marks cannot realistically be negative, there are no lower outliers in this dataset.

4

Find Outliers

df[

    (df['placement_exam_marks'] < lower_limit)
]

This returned all rows considered outliers.

Techniques

Handling Outliers

Method 1: Trimming

Remove outlier rows

trimming.py
new_df = df[
    (df['placement_exam_marks'] <= upper_limit) &
    (df['placement_exam_marks'] >= lower_limit)
]

Original

1000

995

The distribution changed only slightly, becoming a little flatter at the upper end.

Method 2: Capping (Winsorization)

✦ Frequently preferred

Instead of deleting rows, replace extreme values with the nearest limit.

capping.py
import numpy as np

df['placement_exam_marks'] = np.where(
    df['placement_exam_marks'] > upper_limit,
    upper_limit,
    np.where(
        df['placement_exam_marks'] < lower_limit,
        lower_limit,
        df['placement_exam_marks']
    )
)

This keeps the number of rows unchanged while reducing the influence of outliers.

Pros

Advantages of the IQR Method

Does not assume normality.

Robust to extreme values.

Easy to compute and interpret.

Widely used in exploratory data analysis (EDA).

Standard method behind box plots.

Cons

Limitations

May classify valid rare observations as outliers.

Less effective for multimodal distributions.

For very small datasets, quartile estimates may be unstable.

Does not account for relationships between multiple features (multivariate outliers).

Key Interview Questions

Click each question to reveal the answer.

An outlier is a data point that is significantly different from the rest of the observations. It can occur due to human errors, measurement errors, fraudulent activities, or genuine rare events. Not every outlier should be removed — some carry valuable information.
IQR is the difference between the 75th percentile (Q3) and 25th percentile (Q1) of a dataset. It measures the spread of the middle 50% of the data: IQR = Q3 − Q1.
First, find Q1 (25th percentile) and Q3 (75th percentile) using .quantile(0.25) and .quantile(0.75). Then subtract: IQR = Q3 − Q1.
John Tukey introduced 1.5 as a practical rule for identifying mild outliers. It works well across many skewed distributions without assuming normality. For extreme outliers, 3 × IQR can be used instead.
Use IQR when the data is skewed or non-normal. Z-score assumes normality and uses mean/SD which are sensitive to outliers. IQR uses quartiles (median-based), making it robust to skewness and extreme values.
Trimming removes outlier rows entirely (data loss). Capping (Winsorization) replaces extreme values with the boundary limits without deleting any rows. Capping is generally preferred as it preserves dataset size.
Q1 = 25th percentile (25% of data below), Q2 = 50th percentile = Median (50% below), Q3 = 75th percentile (75% below). They divide the dataset into four equal parts.
A box plot shows Q1, median, and Q3 as a box. Whiskers extend to Q1 − 1.5×IQR (lower) and Q3 + 1.5×IQR (upper). Any individual points plotted beyond the whiskers are considered outliers.
IQR is based on quartiles, which are percentile-based measures. Unlike mean and standard deviation, quartiles are not heavily affected by extreme values — changing the maximum value doesn't change Q1 or Q3.
Advantages: No normality assumption, robust to outliers, easy to compute, standard behind box plots.
Limitations: May flag valid rare data, less effective for multimodal distributions, unstable on very small datasets, doesn't handle multivariate outliers.
Final Notes

Final Takeaways

✔️

Use IQR for skewed (non-normal) numerical data.

✔️

Compute Q1 (25%) and Q3 (75%).

✔️

Calculate IQR = Q3 − Q1.

✔️

Determine the Lower Limit = Q1 − 1.5 × IQR and Upper Limit = Q3 + 1.5 × IQR.

✔️

Values outside these limits are treated as outliers.

✔️

You can either Trim (remove the rows) or Cap/Winsorize (replace extreme values with the limits).

✔️

Always visualize your data (histogram, KDE plot, box plot) and understand the business context before removing or modifying outliers, because some extreme values may represent important real-world events rather than errors.
