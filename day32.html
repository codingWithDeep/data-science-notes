Day 32 — Encoding Numerical Features: Discretization, Binning, and Binarization

This lecture continues the Feature Engineering series. In previous lectures, we learned how to encode categorical features. In Day 32, the focus is the opposite: converting continuous numerical features into discrete categories or binary values.

1. Encoding Numerical Features

Normally, numerical data is already suitable for machine-learning models. However, in some problems, representing numerical values as categories may provide a simpler and more meaningful representation.

For example, suppose a Google Play Store dataset contains the number of app downloads:

AppNumber of Downloads
App A500
App B15,000
App C1,500,000
App D100,000,000

Instead of using the exact download numbers, we can create categories:

Number of DownloadsDownload Category
0–1,000Very Low
1,001–10,000Low
10,001–100,000Medium
100,001–1,000,000High
More than 1 millionVery High
Numerical data should not always remain numerical. Depending on the problem, converting numerical values into categories may produce a more useful representation.

The two major techniques discussed in this lecture are:

  1. Discretization or Binning
  2. Binarization

2. What Is Discretization?

Discretization is the process of converting continuous numerical values into a limited number of discrete intervals. It is also known as: Binning, Bucketing, or Interval discretization.

Suppose we have the following values: 1, 3, 5, 7, 9, 12, 15, 18, 21, 25, 28. We can divide the complete numerical range into intervals:

Bin 1 → 0–10 Bin 2 → 10–20 Bin 3 → 20–30
Original ValueBin
3Bin 1
7Bin 1
12Bin 2
18Bin 2
21Bin 3
28Bin 3

The process is similar to constructing a histogram. Therefore, discretization can be understood as converting continuous numerical values into histogram-like intervals.

3. Why Do We Use Discretization?

Benefit 1: Reduces the Effect of Outliers

Suppose most salaries are between ₹20,000 and ₹100,000, but one person earns ₹10,000,000. After binning, the exact extreme value is masked by the "High-income bin". The effect of the outlier may be reduced. (Note: binning does not remove the outlier, it only reduces the importance of its exact magnitude).

Benefit 2: Can Improve the Distribution of Values

Some binning techniques can distribute observations more evenly, which may improve the representation of highly skewed numerical features.

Benefit 3: Captures Nonlinear Relationships

Age groups may represent a relationship with insurance risk more effectively than exact ages (e.g., 0–18 Low, 19–35 Medium, 36–60 High, Above 60 Very High).

Benefit 4: Improves Interpretability

Compare: Age coefficient = 0.034 vs. "People aged 60 or above have a higher predicted risk." The second explanation is easier for business stakeholders to understand.

4. Types of Discretization

A. Unsupervised Binning

The target variable is not used while creating bins. The lecture discusses three techniques:

  1. Equal-width or uniform binning
  2. Equal-frequency or quantile binning
  3. K-Means binning

B. Supervised Binning

The target variable is used while selecting the bin boundaries. An example is Decision-tree-based binning. (Mentioned but not explained in detail).

C. Custom or Domain-Based Binning

Bins are manually created using domain, business, or industry rules. Example: Age 0–17 → Child, Age 18–59 → Adult, Age 60+ → Senior citizen.

5. Equal-Width or Uniform Binning

Equal-width binning divides the complete numerical range into intervals having the same width.

Bin Width = (Maximum Value - Minimum Value) / (Number of Bins)

Suppose: Min = 0, Max = 100, Number of bins = 10. Then: Width = 10.

Bin 1 → 0–10 Bin 2 → 10–20 ... Bin 10 → 90–100

Advantages

  • Simple to understand and calculate
  • Reduces the effect of extreme values
  • Useful when values are relatively uniformly distributed

Limitations

Major Flaw: Equal-width binning does not guarantee an equal number of observations in every bin. For skewed data, you might get 900 observations in Bin 1, and only 1 observation in Bin 5. It generally does not improve the value spread of highly skewed features.

6. Equal-Frequency or Quantile Binning

Equal-frequency binning attempts to place approximately the same number of observations in every bin. It is also called Quantile binning.

Suppose we want ten bins. The data is divided using 10th, 20th, 30th... 100th percentiles. Each bin contains approximately 100% / 10 = 10% of the observations.

BinNumerical RangeNumber of Observations
Bin 10–16Approximately 100
Bin 216–20Approximately 100
Bin 320–22Approximately 100
PropertyEqual WidthQuantile
Bin widthsEqualUsually different
Number of observationsMay be very differentApproximately equal
Handles skewed dataLimitedUsually better
Limitation: If many observations have exactly the same value, it may not be possible to create perfectly equal-frequency bins because quantile boundaries may become identical.

7. K-Means Binning

K-Means binning uses the K-Means clustering algorithm to identify groups in numerical data. It is useful when observations naturally occur in clusters with large empty gaps between them.

How K-Means Works

  1. Initialize: Select K initial cluster centres (centroids) randomly.
  2. Calculate Distances: Measure distance between every observation and every centroid.
  3. Assign: Assign every observation to its nearest centroid.
  4. Update: Calculate new centroids as the mean of all observations in the cluster.
  5. Repeat: Stop when assignments or centroids no longer change significantly.
When to use: When data naturally forms clusters, large gaps exist between groups, or equal-width/quantile boundaries divide natural groups incorrectly.
Limitations: More computationally expensive, results depend on initialization, and it is not useful when data has no natural clusters.

8. Encoding the Discretized Variable

After creating bins, the resulting categories must be encoded.

A. Ordinal Encoding

Bins are represented using ordered integers: Bin 1 → 0, Bin 2 → 1, Bin 3 → 2. This is generally appropriate because numerical intervals naturally have an order (e.g., Low income < Medium income < High income).

B. One-Hot Encoding

Every bin is converted into a separate binary column. This may be useful when the relationship between bins is not expected to be monotonic, but ordinal encoding is more commonly used for naturally ordered intervals.

10. Simple Code Example

import numpy as np
from sklearn.preprocessing import KBinsDiscretizer

X = np.array([[5], [10], [15], [20], [30], [50], [100]])

discretizer = KBinsDiscretizer(n_bins=3, encode="ordinal", strategy="quantile")

X_binned = discretizer.fit_transform(X)
print(X_binned)
# Possible Output: [[0.] [0.] [0.] [1.] [1.] [2.] [2.]]

Viewing Bin Boundaries

print(discretizer.bin_edges_)
# Possible Output: [array([ 5., 15., 30., 100.])]

Intervals: Bin 0 → 5–15, Bin 1 → 15–30, Bin 2 → 30–100.

11. Titanic Dataset Example

The lecture uses the Titanic dataset and selects numerical columns such as Age and Fare. Target: Survived.

Applying Binning Using ColumnTransformer

from sklearn.compose import ColumnTransformer
from sklearn.preprocessing import KBinsDiscretizer

age_discretizer = KBinsDiscretizer(n_bins=10, encode="ordinal", strategy="quantile")
fare_discretizer = KBinsDiscretizer(n_bins=10, encode="ordinal", strategy="quantile")

transformer = ColumnTransformer(
    transformers=[
        ("age_binning", age_discretizer, ["Age"]),
        ("fare_binning", fare_discretizer, ["Fare"])
    ]
)

X_train_transformed = transformer.fit_transform(X_train)
X_test_transformed = transformer.transform(X_test)
Important: Fit the transformer only on the training dataset. Do not fit it separately on the testing dataset because that can cause data leakage.

12. Important Lesson About Accuracy

Feature engineering does not guarantee better accuracy. A transformation may improve, reduce, or produce almost no change in accuracy. The usefulness depends on the dataset, feature distribution, model, number of bins, and binning strategy. Always compare performance using cross-validation.

13. Choosing the Number of Bins

The number of bins is a hyperparameter.

  • Too few bins may remove useful information (e.g., grouping ages 0–40 into one bin).
  • Too many bins may provide very little simplification (e.g., Age 20 → Bin 20, Age 21 → Bin 21 behaves almost like the original feature).

Test different values: n_bins = [3, 5, 7, 10, 15] and select using cross-validation.

15. What Is Binarization?

Binarization converts numerical values into only two possible values: 0 or 1. It uses a threshold.

x_binary = 0    if x ≤ threshold
x_binary = 1    if x > threshold

Income Example

Annual income ≤ ₹600,000 → 0 (Not taxable). Annual income > ₹600,000 → 1 (Taxable).

Image-Processing Example

Grayscale image pixels (0 to 255) converted to black and white using a threshold of 127.5. Pixel ≤ 127.5 → 0 (Black). Pixel > 127.5 → 1 (White).

16. Scikit-Learn Binarizer

from sklearn.preprocessing import Binarizer
import numpy as np

X = np.array([[2], [5], [10], [15], [20]])

binarizer = Binarizer(threshold=10)

X_binary = binarizer.transform(X)
print(X_binary)
# Output: [[0] [0] [0] [1] [1]]

Values less than or equal to 10 become 0. Values greater than 10 become 1.

Important Parameters

  • threshold: Defines the dividing value.
  • copy: If True, preserves original input. If False, modifies data directly.

19. When Should Discretization Be Used?

Consider when: Distribution is highly skewed, extreme values have excessive influence, natural groups exist, business rules define intervals, or interpretability is important.

Avoid when: Exact numerical differences are important, the model handles non-linearity well, or binning would remove important information.

20. Important Data-Science Considerations

Information Loss

Original: 21, 25, 29. After binning: All → Bin 2. The model can no longer distinguish between them. Discretization trades numerical precision for simplicity.

Artificial Discontinuities

If Bin 1 is Age 0–30 and Bin 2 is Age 31–60, a person aged 30 and a person aged 31 are almost identical but placed in different categories.

Fit Only on Training Data & Use Pipelines

Correct Workflow:
from sklearn.pipeline import Pipeline

pipeline = Pipeline(steps=[
    ("binning", discretizer),
    ("model", model)
])
Pipelines reduce the risk of data leakage.

21. Final Summary

  1. Discretization or binning converts continuous numerical values into intervals.
  2. Equal-width binning creates intervals with equal numerical widths (bad for skewed data).
  3. Quantile binning creates intervals containing approximately equal numbers of observations (good for skewed data).
  4. K-Means binning identifies intervals based on natural clusters.
  5. Custom binning creates intervals using business or domain knowledge.
  6. Scikit-learn provides KBinsDiscretizer with uniform, quantile, and kmeans strategies.
  7. Discretized values can be encoded using Ordinal encoding or One-hot encoding.
  8. Binarization converts numerical values into 0 or 1 using a threshold via Binarizer.
  9. Binning may improve interpretability, robustness to extreme values, and skewed distributions, but it may also cause information loss, artificial boundaries, and reduced accuracy.

The correct strategy and number of bins should be selected using cross-validation and domain knowledge. 📊
