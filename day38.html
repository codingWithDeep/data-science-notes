Random Sample Imputation

Definition

Random Sample Imputation replaces missing values by randomly selecting existing values from the same column.

Instead of creating new values, we reuse values that already exist in the dataset.

Example

The new values already existed in the dataset. We do NOT generate completely random numbers.

Why not use random numbers like 500 or 900?

Because those numbers may not belong to the actual data distribution.

Instead, we randomly pick from the observed values only.

Advantages

1

Very Easy to Apply

Concept is extremely simple. Only randomly sample from existing observations.

2

Preserves Data Distribution ⭐

This is the biggest advantage. Unlike Mean or Median Imputation, Random Sample Imputation keeps the overall distribution almost unchanged.

Original

*
*****
*********
*************

After Random Sample

*
*****
********
*************

Almost identical.

Why?

Because frequently occurring values have a higher probability of being selected.

Suppose Age 22 appears 200 times and Age 80 appears 5 times.

Random sampling naturally selects 22 much more often than 80. Hence the distribution remains almost the same.

3

Variance is Preserved

Mean imputation reduces variance because every missing value becomes the same mean. Random Sample Imputation preserves variance because different existing values are inserted. This is especially useful for statistical models.

Best For — Linear Models

Linear Regression Logistic Regression Linear SVM

These models assume certain statistical properties of data. Since Random Sample Imputation preserves distribution, it generally performs better.

Not Ideal For — Tree-Based Models

Decision Tree Random Forest XGBoost LightGBM

Because trees split data differently and random replacement may introduce unnecessary noise.

Disadvantages

1

Correlation Gets Disturbed

Although individual column distribution stays similar, relationships between columns change.

Original

Age ↑  →  Salary ↑

Strong correlation

After Random Replacement

Age values shuffled

Correlation becomes weaker

2

Random Noise is Introduced

The inserted values are random. Therefore additional randomness enters the dataset.

3

Memory Problem During Deployment

During prediction, the model needs access to the original training values. Future missing values must also be replaced using values sampled from the training data.

If training data is huge, server memory usage increases.

Production Tip ⭐ — Very Important

Suppose a user submits the same input multiple times. Random sampling every time may generate different imputations.

Input 1 — Age missing

Model inserts 24

Prediction = Approved

Same Input — Second time

Model inserts 41

Prediction changes!

This is undesirable. Different predictions for the same input.

Solution

Use a fixed random seed (random_state) based on a deterministic value (for example, another feature or row identifier).

Same input → Same sampled value → Stable predictions

Random Sample Imputation for Categorical Data

Example — House Prices

FireplaceQu ~50% missing
GarageQual ~5% missing

Randomly sample existing categories: Excellent, Good, Fair, Poor — randomly assign one.

~5% Missing

Category frequencies remain almost unchanged. Technique works well.

~50% Missing

Category frequencies change significantly. This becomes a red flag.

Important Conclusion: Random Sample Imputation works better when missing percentage is relatively low. If missing percentage is very high, choose another imputation technique.

Missing Indicator

One of the most interesting feature engineering techniques.

Idea

Instead of only filling missing values, create a new binary column indicating whether the original value was missing.

Original

Age
22
31
NaN
45

After — Create Age_NA

AgeAge_NA
22False
31False
NaNTrue
45False

Then perform normal imputation on the original column.

Why Does This Help?

Sometimes, missingness itself contains useful information.

Medical Dataset

Patients who skipped a test may differ systematically from those who completed it.

Bank Dataset

People who did not report income may have different credit behavior.

The model can learn this hidden pattern.

Kaggle Competition Note

Participants used Missing Indicator and it significantly improved performance.

Intuition

Machine Learning models can learn: Rows with missing values vs Rows without missing values. This additional information sometimes improves prediction.

Does it Always Work? No. It is not guaranteed. But it is a useful trick worth trying.

Step 1 — Create MissingIndicator

Step 2 & 3 — Fit & Transform

A new boolean column is generated: True = Missing, False = Not Missing

Easier Method — add_indicator=True

Instead of using MissingIndicator separately, Scikit-Learn provides:

SimpleImputer(
  strategy='mean',
  add_indicator=True
)

Advantages

  • Simpler
  • Cleaner
  • No need to merge columns manually

This is the recommended approach in modern Scikit-Learn pipelines.

Automatically Selecting Best Imputation Method

Question

How do we know whether Mean or Median is better? Or Most Frequent vs Constant?

GridSearchCV

GridSearchCV automatically tests different parameter combinations and selects the one giving the best cross-validation performance. Instead of manually trying every imputation strategy, GridSearchCV performs the search automatically.

Pipeline Used

ColumnTransformer

Logistic Regression

Numerical Parameters

Categorical Parameters

Then selects the best-performing combination.

Avoids manual experimentation

Best Practices for Data Scientists ⭐

Use Mean Imputation when

Data is normally distributed · No extreme outliers

Use Median Imputation when

Data is skewed · Outliers exist

Use Most Frequent when

Categorical feature has few missing values

Use Random Sample Imputation when

You want to preserve the original distribution · Missing percentage is relatively low · You are using linear models

Use Missing Indicator when

Missing values may themselves contain information · Performance improvement is desired · Especially useful with linear models and some ensemble methods

Always Evaluate Different Imputation Methods

Never assume one method is universally best. Use Cross-validation, Pipelines, and GridSearchCV to identify the best strategy for your dataset.

Summary Table

Technique Best Use Case Advantages Disadvantages
Mean Imputation Normally distributed numerical data Simple and fast Reduces variance, sensitive to outliers
Median Imputation Skewed numerical data Robust to outliers Slight distortion of distribution
Most Frequent Categorical data Easy and effective May overrepresent dominant category
Random Sample Preserve original distribution Maintains variance and distribution Alters feature correlations, introduces randomness
Missing Indicator Missingness carries information Can improve model accuracy Not beneficial for every dataset
GridSearchCV Selecting best strategy Automated and data-driven Computationally more expensive

Key Takeaways

1

Random Sample Imputation replaces missing values by randomly picking from existing observations — it does not generate new numbers.

2

Its biggest advantage is preserving the original distribution and variance — frequently occurring values are naturally selected more often.

3

It works best for linear models and when missing percentage is relatively low. It is not ideal for tree-based models.

4

Key disadvantages: correlation gets disturbed, random noise is introduced, and memory overhead during deployment (must store training values).

5

Production tip: Use a fixed random_state to ensure the same input always gets the same imputed value — avoiding unstable predictions.

6

Missing Indicator creates a new binary column (True/False) showing where values were originally missing — letting the model learn patterns from the missingness itself.

7

The easiest way to use Missing Indicator: SimpleImputer(add_indicator=True) — the recommended modern approach.

8

GridSearchCV automates the search for the best imputation strategy by testing all parameter combinations (e.g., Mean+MostFrequent, Median+Constant, etc.) via cross-validation.

9

Never assume one method is universally best. Always evaluate using Cross-validation, Pipelines, and GridSearchCV to find the right strategy for your specific dataset.
